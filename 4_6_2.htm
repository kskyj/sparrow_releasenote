Eclipse Plugin  개선

 

 

- 파일수집/NEST업로드 프로그레스 세분화

파일수집, NEST 업로드 단계에서 프로그레스를 세분화 하였습니다.

 

 

- 라이선스 메시지 수정

라이선스 초과시 제공하는 메시지를 수정하였습니다.

 

 

- Eclipse Plugin properties 에 따라 옵션 뷰설정 및 동작 제어 개발

플러그인에 프로퍼티를 설정하고, 해당 옵션에 따라 동작 함. 위치는 plugins 폴더안의 jar 파일 압축 해제하고, conf 안의 sparrow.eclipse.properties 파일을 수정

 

nest.upload.show.option=true --> 분석 후 네스트 업로드 할지 보이는 옵션

interclass.level.show.option=true --> 인터클래스 레벨 조절 보이는 옵션 설정

 

#Default value when options are false

nest.upload.default.value=true --> 위의값이 false 일때 기본값

interclass.level.default.value=1 --> 위의 값이 fasle 일때 기본값

 

 

- 자동 로그인 항상 활성화 되도록 변경

로그인 여부와 관계없이 자동로그인 체크박스 항상 활성화 되도록 변경

 

 

엔진 개선

 

 

- JSP 분석시 전용 체커만 분석하도록 변경

JSP 분석시 아래 체커들만 분석하도록 변경함. 즉 JSP는 아래체커 + PMD 만 동작함

XSS

XSS_DOM

EVAL_INJECTION

CROSS_SITE_REQUEST_FORGERY

EXPOSURE_OF_ADMINISTRATION_PAGE

MISSING_AUTHENTICATION_CONTROL

 

 

- 소스파일 인코딩 자동 감지하는 기능 추가

분석시 소스코드의 인코딩을 자동으로 감지하여 최대한 정확한 결과가 나오도록 변경

 

 

- Mingw 로 컴파일한 ocaml 엔진 적용(opus1만 적용)

분석중 멈추는 에러를 없애기 위해 적용함

 

 

기타 개선

 

 

- rmi 에러시 로그 수집하도록 개선

rmi를 통한 분석시 에러가 발생할 경우 로그를 수집하게 변경하였습니다.

 sparrow4/logs 폴더에 파일 생성

 upload-failed-whistle.log : 에러 발생한 시점의 sparrow-whistle.log를 백업해둠 20개까지

 upload-failed-engine-client.log : 엔진 클라이언트 upload 메소드 진행 상황을 상세히 기록함. 에러가 아니면 파일 기록안함

 

 

- Semantic 분석시 summary 파일 처리 개선

semantic engine이 전분석단계에서 summary 파일을 생성시 디렉토리 세분화 해서 생성하였습니다.

 

 

- analysis 옵션으로 분석시 기존 map_files 폴더 삭제

analysis 옵션으로 sparrow 폴더 분석시 기존 map_files 폴더를 삭제하도록 변경하였습니다.

 

 

- inter_class_level 0 일때 인터클래스 동작(pre) 하지 않도록 변경

 

 

- extract 할때 파일 개수 보이도록 프로그레스바 세분화

- 정확도 개선 및 스펙 변경

* NULL_CHECK_AFTER_DEREFERENCE 정확도 개선

 

* BAD_RETURN_TEST

1. C++의 boolean type을 반환하는 함수에 대해서는, if 구문 안에서 함수를 호출한 뒤 결과 값을 명시적으로 상수와 비교하지 않더라도 오류를 검출하지 않도록 변경

2. 함수의 RETURN 규약이 너무 다양하여(0이 성공일 경우도 있고, 음수로 반환되는 반환 값을 범위로 잘라(예: -100 ~ -200) 오류 처리에 사용하는 경우도 있습니다.)

함수의 반환값에 대해 비교하지 않는 경우만 검출하는 내용으로 변경

 

* DO_NOT_USE_SIZEOF_ON_PTR

현재 포인터(*) 형으로 명시적으로 선언된 변수가 사용될 때에만 오류를 검출합니다.

그러나 다음과 같이 & 연산자 등으로 일반 변수를 지정해 사용하는 경우도 검출하도록 수정

typedef struct __ST_SPR {

   char    s_v1[10];

   char    s_v2[20];

   int     i_len;

} ST_SPR;

ST_SPR    st_sparrow;

memset((void *)&st_sparrow, 0, sizeof(&st_sparrow));

 

* MISSING_VALIDATION_OF_ARRAY_INDEX

Unsigned char 타입의 변수일 경우 0-255 의 값을 가진다고 가정하고 분석,

255보다 작은 사이즈의 배열에 대해 Unsigned char 타입의 인덱스로 접근하는 경우 알람을 생성하지 않음

아래와 같은 false alarm 제거

If(seq < MAX) {

If(buf[seq].settle) <- 이부분 검출 안됨

 

 

* DO_NOT_USE_DEPRECATED_FUNCTIONS

최신 CERT를 반영하여, 검출 대상 함수를 11개로 축소하였습니다.

기존에 문제가 되던 memcpy 함수는 제외되었습니다.

 

* BAD_OPERATOR.ASSIGNMENT_IN_COND

BAD_OPERATOR.ASSIGNMENT_IN_COND의 경우, 바깥에 괄호가 있으면

무조건적으로 검출하지 않도록 합니다.

또한 assignment의 대상이 함수 호출인 경우 검출하지 않도록 합니다.

검출되지 않는 경우의 예시는 아래와 같습니다.

if((a = b)) { ... }

if(!(a = func())) { ... }

 

 

 

NEST 화면 개선

 

 

- 체커그룹, 결함목록에 CWE/CERT/시큐어코딩가이드 컬럼 표시

checker reference 메뉴를 이용하여 CWE/CERT/OWASP/시큐어 코딩가이드 매핑을 체커그룹 결함 목록에서 볼 수 있습니다.

*체커 그룹

 

* 결함 목록

 

 

- 체커 그룹 이름 변경 기능

체커 그룹의 이름을 변경 할 수 있습니다. 이름 변경시 프로젝트 페이지에서는 변경한 이름의 체커그룹으로 수정되며, 스캔 페이지의 경우 기존에 분석했던 이름으로 남아 있습니다.

스캔페이지의 체커그룹 클릭시에는 analysis detail 화면으로 넘어갑니다.

 

 

- 제외 함수 설정 기능 추가

결함목록에 업데이트 하지 않을 함수 목록 지정이 가능합니다. 와일드(*)형식을 지원합니다.

해당리스트의 항목은 분석은 하나, 결함목록에 업데이트 하지 않습니다.

 

- 프로젝트, 스캔 화면에 분석자, 버전, 체커그룹, 설명 정보 표시

프로젝트와 스캔 페이지에

1. 프로젝트 생성,수정시 적은 Memo를 표시합니다.

2. 분석당시의 체커그룹을 표시합니다. 단 프로젝트 페이지는 현재 지정된 체커그룹을 표시합니다.

3. 분석시 -U 를 통해 입력한[whistle manager의 경우 로그인한] 사용자를 표시 합니다.

4. 스캔페이지의 analysis detail 메뉴를 통해 분석 당시의 SPARROW version 을 표시합니다.

 

 

- 로그인 페이지 개선

1. ID/Password 표시 네모상자에서 표시

2. save id 위치 변경

3. login 버튼 변경

4. sparrow version , 저작권 표시

 

- 페이징 번호 개선

페이지 번호가 시인성, 클릭하기 쉽도록 변경 되었습니다.

 

- 사용자 일괄 변경 및 검색 기능 추가

사용자 관리 페이지에 검색기능과 일괄 정보 변경가능하도록 수정되었습니다.

 

- PMD 체커 화면 제거

PMD 체커화면이 제거 되었습니다.

 

- 히스토리 페이지 제거된 결함 항목 제거

히스토리 페이지의 항목중 [제거된 결함] 이 제거 되었습니다.

 

- 세션 자동 타임 아웃 기능 추가

NEST 에서 1시간 동안 아무 작업 없을시 자동으로 세션 로그아웃 되는 기능을 추가 하였습니다.

시간은 setup 시 조절 가능 합니다.

Class Hierarchy 와 Checker 를 분석할 때마다 로드했지만, 한번만 하도록 변경하였습니다.

 

- NEST Project 접속 속도 개선

- 전체 분석 속도 개선

중복 체커 제거하는 과정을 기존 db에서 하지않고 중복체커가 필요한 것만 선별해서 메모리에서 처리하였습니다.

 

 

 

기타 개선

 

 

- 로그 세분화

분석 후 sparrow/log_files/summary.log 에서 분석명령어, 분석결과, 각 단계별 분석 시간, 엔진 버전을 표시합니다.

 

- .pc 파일 처리시 .c 업로드 안함

proc 파일 분석시 동일한 이름의 .c 파일은 업로드 하지 않습니다.

 

- 룰체커 분석시 .i 와 .c 같이 분석

매크로 체커는 기본적으로 .c 로 들어가게 변경하였습니다.

 

- pmd xml 1000개단위로 병합

DISK I/O 를 위해 PMD 체커 개수를 1000개 단위로 병합하였습니다.

 

- setup 시 heap 메모리 자동설정

setup 시 Heap 메모리가 시스템의 30%로 자동으로 설정되며, setup을 통해 변경가능합니다.

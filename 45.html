- JSP / Java Script 분석 방법 개선

Java 를 제외한 JSP / Java Script 들은 알람이 거의 검출되지 않는 Semantic 과 Java 체커는 제외하고, PMD 로만 분석하고 무조건 Java 와 함꼐 분석 합니다.

이번 버전부터  JSP, JS 를 분석할떄 jsp 를 컴파일 하지 않아 속도가 느려지지 않습니다.

하지만 PMD extension 미지원으로 .jsp / .js 파일만 분석 됩니다.

분석 방법 개선

- JSP/JS 무조건 분석

jsp/js 체커가 활성화 되있을 경우 sparrow-java명령어를 실행하면 빌드 path 밑에 .jsp / .js 를 검색하여 무조건 분석 합니다.

즉 JSP/JS 분석을 위한 필요한 옵션이 없습니다.


- 마지막 결과 삭제하고 신규 결과로 대치하는 옵션 추가

-R / --replace 옵션을 사용하면 scan 을 늘리지 않고 마지막 scan 을 지우고 대치합니다. 자동 분석으로 자주 분석 하는 시스템에 적합 합니다.


- BD 옵션 optdir 파일로 변경

기존 edg로 전처리를 하기 위해 지정한 -builin_prep 옵션은 optdir 파일로 변경 되었습니다. 또한 명칭이 built_prep --> sparrow_prep 로 변경되었습니다.

휘슬매니저에서 시스템 헤더를 지정하면 자동으로 처리됩니다.


- edg로 전처리 하는것 Preset에 추가

edg로 전처리 하는것이 preset에 추가되어 자동으로 휘슬 매니저에서 sparrow prep 옵션 선택 창이 나옵니다.

현재 Calmshine, PM+ 는 아래와 같이 옵션이 자동으로 추가됩니다.


- 분석 제외 리스트 네스트에서 관리

기존 optdir 로 관리하던 분석 제외 목록은 NEST 의 프로젝트 에서 관리하게 변경 하였습니다.


- 전체 프로젝트에 적용 되는 제외 리스트

NEST프로젝트 관리 외에 전체 제외 목록을 제공합니다.

sparrow4/whistle/engine/conf/default_exclude_list.txt 를 수정하면 전체 프로젝트에 적용 됩니다.

기본값으로 아래와 같이 있습니다.

- 업데이트 안하는 옵션

-NU / --noupdate 옵션을 사용하면 업데이트 체크를 하지 않습니다. prep/extract 시에는 자동으로 업데이트 체크를 하지 않습니다.


- 빌드 검사 안하는 옵션

이 옵션 사용시 강제로 분석 합니다.
